Reports use of gamepad1 or gamepad2 inside an OpMode constructor or class initializer.

Controller input should be read during the OpMode loop, not during object construction or static initialization.

Why it matters: During construction/initialization the OpMode hasnâ€™t started and gamepad state may be uninitialized or stale. Reading it there can give wrong values or crash later assumptions.

Triggered when: A constructor or class initializer in a class recognized as an OpMode references gamepad1 or gamepad2.

Quick fix: Delete statement removes the offending statement. Move input handling into runOpMode() (LinearOpMode) or loop() (Iterative).

Bad:

public class Drive extends LinearOpMode {
    boolean slow = false;
    public Drive() {
        slow = gamepad1.left_bumper; // not safe here
    }
    @Override public void runOpMode() { /* ... */ }
}

Good:

public class Drive extends LinearOpMode {
    boolean slow = false;
    @Override public void runOpMode() {
        waitForStart();
        while (opModeIsActive()) {
            slow = gamepad1.left_bumper; // read during loop
            telemetry.update();
        }
    }
}
