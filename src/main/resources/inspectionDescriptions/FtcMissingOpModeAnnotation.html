Reports a public OpMode class that lacks an @TeleOp or @Autonomous annotation.

Without an OpMode annotation, the class wonâ€™t appear in the Driver Station list.

Why it matters: FTC discovers OpModes via annotations. A missing annotation hides your code from the Driver Station menu.

Triggered when: A public class recognized as an OpMode has no @TeleOp or @Autonomous annotation.

Quick fix: Add @TeleOp annotation inserts @TeleOp(name="<ClassName>") above the class. You can change it to @Autonomous later if needed.

Bad:

public class Drive extends LinearOpMode {
    @Override public void runOpMode() { /* ... */ }
}

Good:

@com.qualcomm.robotcore.eventloop.opmode.TeleOp(name = "Drive")
public class Drive extends LinearOpMode {
    @Override public void runOpMode() { /* ... */ }
}
