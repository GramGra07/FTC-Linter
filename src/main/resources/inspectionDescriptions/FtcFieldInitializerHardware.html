Reports use of hardwareMap in a field initializer of an FTC OpMode class.

Hardware lookups must run after the OpMode is initialized, not at field declaration time.

Why it matters: During class construction, hardwareMap may be null or not ready; referencing it in initializers can crash or lead to undefined behavior.

Triggered when: A field declared in a class recognized as an OpMode has an initializer that references hardwareMap (for example, hardwareMap.get(...) or hardwareMap.something).

Quick fix: Remove hardwareMap initializer clears the initializer. Move initialization into runOpMode() (LinearOpMode) or init() (Iterative OpMode).

Bad:

public class Drive extends LinearOpMode {
    DcMotor lf = hardwareMap.get(DcMotor.class, "lf"); // runs too early
    @Override public void runOpMode() { /* ... */ }
}

Good:

public class Drive extends LinearOpMode {
    DcMotor lf;
    @Override public void runOpMode() {
        waitForStart();
        lf = hardwareMap.get(DcMotor.class, "lf");
        while (opModeIsActive()) {
            // ...
        }
    }
}
