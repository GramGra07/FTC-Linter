Reports telemetry.addData(...) calls in an OpMode method when that method never calls telemetry.update().

Without telemetry.update(), added lines are not sent to the Driver Station and wonâ€™t be visible.

Why it matters: FTC telemetry buffers data; update() must be called to flush to the Driver Station UI.

Triggered when: A method in an OpMode contains telemetry.addData(...) but no telemetry.update() anywhere in the same method.

Quick fix: Insert telemetry.update() tries to add the call right after the offending addData(), or appends it at the end of the method as a fallback.

Bad:

public void runOpMode() {
    telemetry.addData("status", "init");
    waitForStart();
    telemetry.addData("running", true); // no update()
}

Good:

public void runOpMode() {
    telemetry.addData("status", "init");
    telemetry.update();
    waitForStart();
    telemetry.addData("running", true);
    telemetry.update();
}
