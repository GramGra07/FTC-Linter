Reports a while-loop nested inside a while(opModeIsActive()) loop in an FTC OpMode.

Nesting blocking loops inside the active loop can freeze updates and prevent cooperative multitasking; use an if or refactor to a single loop.

Why it matters: The FTC event loop expects short iterations. Inner while-loops can block iteration, causing delayed telemetry and watchdog issues.

Triggered when: A while-statement appears within another while whose condition contains opModeIsActive().

Quick fix: Replace inner while with if rewrites the inner while(...) { ... } as if (...) { ... }.

Bad:

public void runOpMode() {
    waitForStart();
    while (opModeIsActive()) {
        while (arm.isBusy()) { // nested while
            // busy-wait
        }
        telemetry.update();
    }
}

Good:

public void runOpMode() {
    waitForStart();
    while (opModeIsActive()) {
        if (arm.isBusy()) {
            // do a little work, then let the loop iterate
        }
        telemetry.update();
    }
}
