Reports a LinearOpMode.runOpMode() method that never calls waitForStart().

Linear OpModes must call waitForStart() to block until the Driver Station presses START.

Why it matters: Without waitForStart(), the OpMode may run during INIT and break event loop expectations or hardware timing.

Triggered when: A class recognized as LinearOpMode overrides runOpMode() and the method body has no call to waitForStart().

Quick fix: Insert waitForStart() adds the call before the first if/while that references opModeIsActive(), or inserts it at the top of the method as a fallback.

Bad:

public class MyAuto extends LinearOpMode {
    @Override public void runOpMode() {
        // no waitForStart()
        while (opModeIsActive()) {
            // ...
        }
    }
}

Good:

public class MyAuto extends LinearOpMode {
    @Override public void runOpMode() {
        waitForStart();
        while (opModeIsActive()) {
            // ...
        }
    }
}
